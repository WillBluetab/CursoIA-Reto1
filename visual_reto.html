Requisitos Funcionales

Router de intención

Un nodo clasifica la petición en una de tres intenciones (reconciliación / casual / romántico) usando salida estructurada (Pydantic) para ajustar el tono.

Agente ReAct con mínimo 4 tools tipadas

Define contratos estrictos con Pydantic. Ejemplos: analizar_perfil(), sugerir_plan() (catálogo ≥ 8 entradas), auditar_respeto() y estimar_longitud().

Bucle Evaluator-Optimizer

Un nodo evaluador puntúa el borrador (0-10) usando una rúbrica con 4 criterios. Si la puntuación promedio es inferior al umbral, el agente reescribe usando el feedback (máximo N vueltas).

Guardrails Duros en código

El código de tu aplicación DEBE rechazar el borrador y forzar el reintento si auditar_respeto devuelve aprobado=False. La seguridad no se delega enteramente al prompt del LLM.

Trazabilidad e historial

El estado del grafo debe guardar el historial de las iteraciones (borradores, puntuaciones, feedbacks) utilizando un reductor (reducer) en LangGraph.

Requisitos Técnicos (Calidad del Código)

Modularidad estricta

Separar de forma rigurosa en archivos: tools.py, prompts.py, schemas.py, graph.py, settings.py, y main.py.

Tipado Estricto (Typing)

Todas las firmas de funciones deben estar completamente tipadas. No usar Any a menos que sea justificado mediante comentarios.

Pydantic para Contratos

Uso estricto de Pydantic con validadores avanzados (ej. Field con descripciones y constraints como ge, le).

Manejo de Errores y Robustez

Las herramientas deben atrapar excepciones (ej. problemas de clave o formato) y devolver mensajes estructurados al agente en lugar de romper el grafo.

Cortafuegos (Recursion Limit)

Definir y documentar un recursion_limit seguro en la ejecución del grafo para prevenir bucles infinitos de coste descontrolado.

Testing Automatizado con pytest

Tests unitarios deterministas para las tools (sin llamar a LLM), test de compilación del grafo y test del guardrail duro (comprobar que un mensaje irrespetuoso es bloqueado).

Entorno Autónomo (uv)

El proyecto debe usar uv. Debería bastar con correr uv sync y uv run langgraph dev para arrancar todo sin configuraciones manuales extra.

Sin emojis

No utilices emojis en el código ejecutable ni en la documentación principal (por estándares de código limpio de producción).

Pistas & Guías de Implementación

¿Cómo implementar el Guardrail Duro?

El guardrail no debe ser parte de la lógica interna del LLM. Impleméntalo en una arista condicional (conditional edge) en LangGraph:

  • El agente produce un borrador del mensaje.
  • El siguiente nodo corre auditar_respeto(mensaje).
  • En la arista condicional, si aprobado es False, rediriges el flujo de regreso al agente pasándole los motivos de rechazo estructurados para que modifique el tono.

Integración de ReAct con el Evaluador

Visualiza el flujo en dos capas:

  • Capa Interna: Un sub-grafo o nodo de tipo ReAct ejecuta el bucle de razonamiento y herramientas para generar una propuesta.
  • Capa Externa: El evaluador recibe la propuesta terminada y decide si se acepta (pasa al fin del flujo) o se rechaza por baja calidad, inyectando feedback al contexto para una nueva ronda del agente.

Pruebas unitarias sin coste de tokens

Para probar que el grafo funciona sin consumir tokens en tu pipeline de testing:

  • Utiliza los mocks de unittest.mock o herramientas integradas en pytest para simular las llamadas al LLM.
  • Verifica que la salida de la arista condicional tome la decisión correcta dada una entrada preestablecida de aprobación/rechazo.

Extensiones recomendadas si terminas rápido

  • Memoria persistente: Configura un SqliteSaver para soportar múltiples turnos vinculados a un thread_id.
  • Integraciones reales: Reemplaza el mock de sugerir plan por una búsqueda en vivo de lugares o eventos reales utilizando la API de Tavily.
  • Monitoreo de costes: Suma y visualiza la cantidad de tokens de entrada y salida consumidos en cada ciclo.
